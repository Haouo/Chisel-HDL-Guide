# Part 8 — 驗證與除錯基礎建設（assert / cover / difftest / harness）

對應：書 §13 ｜ `import chisel3._`、`import chisel3.util._`

> 大型 Chisel 設計的關鍵不是「能跑」，而是「能持續驗證」。逐步累積自己的 verification vocabulary。

---

## M8.1 `assert` / `cover` 是大型設計必備

```scala
// protocol invariants
assert(!(io.enq.fire && full),  "enq when full")
assert(!(io.deq.fire && empty), "deq when empty")
// cache invariant
assert(PopCount(hitVec) <= 1.U, "multiple hit ways")
// scoreboard invariant
when(io.wb.valid && io.wb.bits.rd =/= 0.U){ assert(busy(io.wb.bits.rd)) }

// coverage：確認重要事件有被測到
cover(io.branchMispredict)
cover(io.cacheMiss)
cover(io.loadUseHazard)
cover(io.storeLoadForward)
```

---

## M8.2 建立 reusable checkers（累積 verification vocabulary）

**一句話**：別到處寫零散 assertion；包成可重用 checker object。

```scala
object CacheChecker {
  def oneHotHit(hitVec: Seq[Bool]): Unit = assert(PopCount(VecInit(hitVec)) <= 1.U)
}
object PipelineChecker {
  def x0AlwaysZero(x0: UInt): Unit = assert(x0 === 0.U, "x0 must be zero")
}
// ready-valid 穩定性 checker（見 Part 3 M3.3）
RVAssert.stableWhenStalled(io.mem.req)
```

## M8.3 除錯訊號：`printf` / `println` / `Printable`

```scala
println(s"Generating cache with ${p.nWays} ways")            // elaboration-time（生成 log）
printf(p"pc=${Hexadecimal(pc)} inst=${Hexadecimal(inst)} rd=${rd}\n")  // simulation runtime
```
> `p"..."`（`Printable`）可內插 Chisel 值並指定格式（`Hexadecimal`/`Decimal`/`Binary`），是 signal formatting 首選。

---

## M8.4 `dontTouch` 不是除錯第一選擇

```scala
dontTouch(debugSignal)        // 臨時：阻止被最佳化掉
```
| 建議優先序 | 手段 |
|-----------|------|
| 1 | 正式 debug IO |
| 2 | Probe / Layer（見 Part 9） |
| 3 | `dontTouch`（臨時） |

⚠️ **陷阱**：太多 `dontTouch` 會讓 generated RTL 變髒、影響最佳化。

---

## M8.5 測試分層（unit → sim → cosim）

| 層 | 工具 | 測什麼 |
|----|------|--------|
| 1 | Scala / ChiselSim unit test | decoder、immgen、ALU、FIFO |
| 2 | Chisel simulation | pipeline handshake、cache FSM、branch predictor |
| 3 | Verilator / C++/Python driver | full CPU、跑 ELF 程式、長時測試 |
| 4 | **ISS differential testing** | RTL commit trace vs ISS golden model |

---

## M8.6 Commit trace 與 differential testing

**一句話**：每拍 commit 一條 architectural state，與 ISS 對拍比對。

```scala
class CommitTrace extends Bundle {
  val valid  = Bool()
  val pc, inst, rdData = UInt(32.W)
  val rdWen  = Bool(); val rd = UInt(5.W); val trap = Bool()
}
```
```
RTL commit one instruction  ─┐
ISS step one instruction    ─┴─► compare pc / rd / rdData / trap
```
> 只比 **architectural** state（Part 6 M6.5）；ROB index 等 micro 細節不比。

---

## M8.7 Simulation-only logic 放 harness，不要塞進 core

| 應放 TestHarness / sim wrapper | 不要放 core RTL |
|-------------------------------|-----------------|
| ELF loader、SDL framebuffer、terminal UART | — |
| ISS cosim、timeout checker、random memory latency model | — |

> core/SoC 是真硬體；harness 是 target environment。混在一起會讓 RTL 無法合成、難移植（FPGA/ASIC）。

---

## M8.8 Full-system regression（CPU 專案）

建議測：
```
rv32ui / rv32mi subset
自寫 bare-metal C tests
cache stress / branch stress
interrupt / timer tests
Snake game smoke test（互動 + framebuffer + UART）
```

**練習**
- ★ 為你的 FIFO 加 `assert(!(enq.fire && full))` 與 `cover(full)`，跑 ChiselSim。
- ★★ 為 Wildcat 拉出 `CommitTrace`，寫一個簡易「golden model（Scala）」對拍比較。
- ★★★ 把 ELF loader 與 timeout 移到 TestHarness，core 只留 memory 介面。
