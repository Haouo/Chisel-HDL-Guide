# Part 5 — 範例設計與互連（FIFO / skid buffer / arbiter / bus）

對應：書 §11、§12 ｜ `import chisel3._`、`import chisel3.util._`

> 整合 Part 1–4 的區塊，做出「中型設計」，當成進入處理器（Part 6）的橋樑。

本章把單點積木接成小系統。FIFO、skid buffer、arbiter、UART、memory-mapped bus 都在處理同一類問題：
資料從一個模組移到另一個模組時，速率、延遲、優先序與 backpressure 如何被明確定義。
這些元件看似 peripheral，但它們其實是 pipeline、cache、memory system 的縮小版。

讀這章時，建議每個設計都畫出「資料何時被接收、何時被保存、何時被送出」三個時間點。
只要這三件事說不清楚，ready/valid 邊界條件就很容易出錯。

**本章路線圖**

- 先學：`Queue`、single-stage buffer、skid buffer、arbiter、UART、簡單 memory-mapped bus。
- 會踩坑：ready critical path、同拍 enq/deq 邊界、讀資料 valid 沒和 data 對齊。
- 最後能做：把多個模組接成有 backpressure、timing isolation 與明確 address decode 的小系統。

## M5.1 FIFO：先用內建 `Queue`

除非教學目的就是練習指標與邊界條件，實務上應該先用 Chisel 內建 `Queue`。它已經處理好
valid-ready、full/empty、`pipe`、`flow` 等常見變體。自刻 FIFO 的價值在於理解行為與驗證邊界，
不是每個專案都重新發明一次。

```scala
class FifoTop(gen: => Data, depth: Int) extends Module {
  val io = IO(new Bundle { val enq = Flipped(Decoupled(UInt(8.W))); val deq = Decoupled(UInt(8.W)) })
  io.deq <> Queue(io.enq, depth)        // 一行搞定
}
```

### Scala 語法：`gen: => Data` 是 by-name parameter

`gen: => Data` 不是 function type，而是 by-name parameter。意思是：每次使用 `gen` 時，右邊那段
產生 `Data` 的 expression 會重新被求值。這在 Chisel 裡常用來避免重用同一個 `Data` object。

```scala
class Wrapper(gen: => Data) extends Module {
  val a = Wire(gen)      // 使用一次 gen，產生一個型別樣板
  val b = Wire(gen)      // 再使用一次 gen，重新產生另一個樣板
}
```

對照：

| 寫法 | 讀法 | 適合 |
|------|------|------|
| `gen: Data` | 呼叫端先建立一個 `Data` 值，再傳進來 | 大多數簡單 generic module |
| `gen: => Data` | 每次用到 `gen` 時重新建立 | 需要多次複製型別樣板的 helper/wrapper |
| `() => Data` | 明確傳一個 Scala function | 較少見，呼叫時要寫 `gen()` |

教學範例若沒有真的使用 `gen`，可以先讀成「這個 module 原本打算支援任意 payload 型別」。
真正寫 generic FIFO/skid buffer 時，請確保 IO 用的是 `gen`，例如 `Decoupled(gen)`，而不是寫死 `UInt(8.W)`。

### 自刻 single-element buffer（bubble FIFO 一格）
```scala
class OneStage(width: Int) extends Module {
  val io = IO(new Bundle { val enq = Flipped(Decoupled(UInt(width.W))); val deq = Decoupled(UInt(width.W)) })
  val full = RegInit(false.B); val data = Reg(UInt(width.W))
  io.enq.ready := !full; io.deq.valid := full; io.deq.bits := data
  when(io.enq.fire){ data := io.enq.bits; full := true.B }
  when(io.deq.fire){ full := false.B }
}
```
| 變體（書 §11.3） | 特性 |
|------------------|------|
| bubble FIFO | 多單元串接，簡單、吞吐低 |
| double-buffer | 兩格交替，吞吐較好 |
| register-memory FIFO | `Reg(Vec)` + 讀寫指標 |
| on-chip-memory FIFO | `SyncReadMem`（大容量省資源） |

對應：§11.1、§11.3。

## M5.2 `Queue` 是 timing 工具，不只是 FIFO

**一句話**：`Queue` 可做 buffering、timing isolation、pipeline elasticity 三種用途。

很多人只把 Queue 當成「存資料的地方」，但在硬體中它更常是 timing 與彈性的工具。
在長 ready path 中插一級 Queue，可以讓上游不必組合看穿很多層下游狀態；在 throughput 需要保持時，
`pipe` 與 `flow` 會影響空/滿邊界下是否能同拍傳遞。

```scala
Queue(io.in, entries = 2)                    // 真 buffering
Queue(io.in, entries = 1, pipe = true)       // 允許同拍 enq+deq（不切吞吐）
Queue(io.in, entries = 1, flow = true)       // 空時可直通（bypass）
```
要理解的旋鈕：`entries` / `pipe` / `flow` / 是否切 ready path。

## M5.3 Skid buffer：打斷 ready critical path

**一句話**：當下游 stall 時多吸一筆，避免 `ready` 變成長組合路徑、又保住吞吐。

Skid buffer 的直覺是「煞車距離」。當下游突然不 ready，上游可能已經送出一筆資料；
skid buffer 用一格暫存吸住這筆資料，讓 ready 不需要一路組合回傳。這種元件很小，
但邊界條件很尖銳，特別是同拍 input fire、output fire、以及 full 清除的優先序。

```scala
class SkidBuffer[T <: Data](gen: T) extends Module {
  val io = IO(new Bundle { val in = Flipped(Decoupled(gen)); val out = Decoupled(gen) })
  val full = RegInit(false.B); val data = Reg(gen)
  io.in.ready  := !full
  io.out.valid := full || io.in.valid
  io.out.bits  := Mux(full, data, io.in.bits)
  when(io.in.valid && io.in.ready && !io.out.ready){ full := true.B; data := io.in.bits }
  .elsewhen(io.out.ready){ full := false.B }
}
```
目標：break ready critical path、preserve throughput、stall 時吸一筆。
⚠️ **陷阱**：simultaneous fire（同拍 in.fire 且 out.fire）邊界條件要仔細測。

## M5.4 Arbiter / RRArbiter

Arbiter 定義的是資源競爭規則。Priority arbiter 簡單、邏輯短，但可能讓低優先權 client 長期 starvation；
round-robin arbiter 較公平，適合共享 memory port、DMA、cache miss pipeline。選哪個不是語法問題，
而是系統層的 service guarantee。

```scala
val arb = Module(new RRArbiter(new MemReq, nClients))   // round-robin（公平）
for (i <- 0 until nClients) { arb.io.in(i) <> client(i).io.req }
mem.io.req <> arb.io.out
```
| `Arbiter` | `RRArbiter` |
|-----------|-------------|
| priority（低編號長期佔優） | round-robin（公平） |

用於：memory system、DMA、I/D-cache arbitration。

## M5.5 UART（baud tick + shift + FSM）

UART 是很好的中型練習，因為它同時包含 timing counter、shift register、FSM 與 Decoupled input。
它也提醒我們：外設常常是「慢速 protocol + 快速系統時脈」的轉換器。baud tick 算錯一點，
短模擬可能看不出來，實際通訊就會慢慢偏移。

```scala
class UartTx(freq: Int, baud: Int) extends Module {
  val io = IO(new Bundle { val tx = Output(Bool()); val channel = Flipped(Decoupled(UInt(8.W))) })
  val BIT_CNT = (freq / baud - 1).U
  val shiftReg = RegInit(0x7ff.U(11.W)); val cntReg = RegInit(0.U(20.W)); val bitsReg = RegInit(0.U(4.W))
  io.channel.ready := (cntReg === 0.U) && (bitsReg === 0.U)
  io.tx := shiftReg(0)
  when(cntReg === 0.U) {
    cntReg := BIT_CNT
    when(bitsReg =/= 0.U){ shiftReg := 1.U ## shiftReg(10,1); bitsReg := bitsReg - 1.U }
    .elsewhen(io.channel.valid){ shiftReg := Cat(Cat(3.U(2.W), io.channel.bits), 0.U(1.W)); bitsReg := 11.U }
  }.otherwise { cntReg := cntReg - 1.U }
}
```
⚠️ **陷阱**：baud tick = `freq/baud` 整數除法有誤差，高 baud 注意 RX 中點取樣對齊。

對應：§11.2。

## M5.6 互連與匯流排（自刻 → 業界標準）

自刻簡單 bus 的教學價值在於讓學生看見 address decode、device select、read latency 對齊與 side effect。
一旦理解這些，再去看 AXI、Wishbone、TileLink 就不會只看到一堆 channel 名稱，而能看出它們在解決
outstanding transaction、ordering、burst、width adaptation 與 clock crossing 等更大的問題。

```scala
// 簡單 memory-mapped 介面（payload 不帶方向，方向交給 master/Flipped）
class MemBus(aw: Int, dw: Int) extends Bundle {
  val addr = UInt(aw.W); val wdata = UInt(dw.W); val rdata = UInt(dw.W)
  val we = Bool(); val re = Bool()
}
// pipelined ack：讀資料延遲對齊
val dataValid = RegNext(io.req && io.re)
```
| 標準 | 特性 | 何時選 |
|------|------|--------|
| Wishbone | 開放、簡單 | 開源 SoC、教學 |
| AXI（AXI4/AXI-Lite） | 高效能、burst、多通道 | ARM 生態、複雜 SoC |
| OCP | 可配置、socket 導向 | 客製互連 |

> Chipyard/Rocket 內部多用 **TileLink**（見 Part 10），外接 DRAM/IP 常用 AXI4，靠 protocol bridge 轉換。

⚠️ **陷阱**：跨匯流排接時，**讀資料延遲**（combinational vs pipelined ack）必須對齊，否則抓到舊資料。

**練習**
- ★ 用 `Queue` 串 UART TX，丟一串 byte 自動送出。完成標準：上游連續 valid 時不會覆蓋尚未送出的 byte。
- ★ 把 `SkidBuffer` 插在兩個會 stall 的 stage 間，量 throughput 不變。完成標準：隨機 ready pattern 下輸出序列與輸入序列相同。
- ★★ 設計 memory-mapped LED/timer 暫存器，bus decoder 路由到不同 device（見 Part 11）。完成標準：讀資料延遲與 ack/valid 對齊。

對應：§11、§12。
