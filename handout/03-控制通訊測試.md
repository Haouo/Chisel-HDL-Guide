## M3.1 有限狀態機（FSM）

```scala
object CacheState extends ChiselEnum { val Idle, CompareTag, WriteBack, Allocate = Value }
val state = RegInit(CacheState.Idle)
switch(state) {
  is(CacheState.Idle)       { when(io.req.valid){ state := CacheState.CompareTag } }
  is(CacheState.CompareTag) { state := Mux(hit, CacheState.Idle, CacheState.Allocate) }
  // ...
}
```

| Moore | Mealy |
|-------|-------|
| 輸出只依 `state`（乾淨、好 timing） | 輸出依 `state`+input（快、但組合路徑長/可能 glitch） |

`ChiselEnum` 適合：cache controller、DRAM controller、replay state、CSR/trap FSM。

⚠️ **陷阱**：Mealy 輸出直拉到 module 邊界要注意 timing；需乾淨輸出就改 Moore 或輸出加一級 register。

## M3.2 流控語意：`Valid` / `Decoupled` / `Irrevocable`（★必懂）

| 介面 | 語意 | 可 backpressure？ | 典型用途 |
|------|------|------------------|----------|
| `Valid(T)` | event / 觀測，不可擋 | 否 | branch update、commit trace、exception |
| `Decoupled(T)` | valid-ready transaction | 是 | memory req、micro-op flow |
| `Irrevocable(T)` | 更強的 Decoupled：valid 後 bits 必須保持穩定到 fire | 是 | 嚴格 producer 契約 |

```scala
val brUpdate = Valid(new BranchUpdate)     // 事件，下游不能擋
val memReq   = Decoupled(new MemReq)       // 可被 backpressure
val commit   = Valid(new CommitTrace)      // 觀測點
```

### Decoupled 通訊與 `.fire`
```scala
class Stage extends Module {
  val io = IO(new Bundle {
    val in  = Flipped(Decoupled(new MicroOp))   // consumer 端要 Flipped
    val out = Decoupled(new ExeResult)
  })
  io.in.ready  := io.out.ready                  // combinational pass-through
  io.out.valid := io.in.valid
  io.out.bits  := execute(io.in.bits)
}
```
> ⚠️ **觀念**：`Decoupled` 只是 **protocol**，**不代表**自動有 register 或 queue。上面是 combinational ready-valid path，**不是** pipeline stage。要真的 buffer/stage：
```scala
val q = Queue(io.in, entries = 2); io.out <> q
```

⚠️ **陷阱**：`valid` 不可組合相依於 `ready`（會形成 combinational loop）。狀態更新一律掛 `.fire`（= `valid && ready`），別只看 `valid`。

對應：§9。

## M3.3 ready-valid 契約用 assertion 守住

**一句話**：「`valid && !ready` 時 payload 必須穩定」做成可重用 checker。

```scala
object RVAssert {
  def stableWhenStalled[T <: Data](x: DecoupledIO[T]): Unit = {
    val stalled     = x.valid && !x.ready
    val prevStalled = RegNext(stalled, false.B)
    val prevBits    = RegEnable(x.bits, stalled)
    when(prevStalled) { assert(x.valid); assert(x.bits.asUInt === prevBits.asUInt) }
  }
}
// 使用
RVAssert.stableWhenStalled(io.mem.req)
```

> 逐步累積自己的 verification vocabulary（見 Part 8）。

## M3.4 Pipeline register 用 `Valid[T]`，bubble 用 `valid := false.B`

**一句話**：別用散亂 registers；把「有效 + payload」包成 `Valid`，bubble 設 invalid。

```scala
// ✗ 散亂
val valid = RegInit(false.B); val pc = Reg(UInt(32.W)); val inst = Reg(UInt(32.W))

// ✓ 包成 Valid
val if_id = RegInit(0.U.asTypeOf(Valid(new FetchPacket)))
when(flush)        { if_id.valid := false.B }     // bubble = invalidate
.elsewhen(!stall)  { if_id := next }
```

> CPU pipeline 插 bubble 用 `uop.valid := false.B`，**不要**塞 NOP 指令。

### Pipeline control 要定義**固定優先序**
```scala
when(kill)         { reg.valid := false.B }
.elsewhen(replay)  { reg := replayPayload }
.elsewhen(!stall)  { reg := next }
```
建議：`kill / flush > replay > stall > advance`。

> CPU bug 常來自「flush 與 stall 同時發生誰優先？exception 與 branch mispredict 同時？interrupt 與 replay 同時？」——把優先序寫死、寫清楚。

⚠️ **陷阱**：`RegInit(0.U.asTypeOf(...))` 方便但「zero encoding 是否合法狀態？enum 的 zero 是否有效？」未必。大型設計用 explicit default（見 M3.5）。

## M3.5 Companion object 建立合法 default / constructor

**一句話**：用 companion object 提供 `bubble` / `fromDecode`，比到處手填欄位乾淨。

```scala
object MicroOp {
  def bubble: MicroOp = { val w = Wire(new MicroOp); w := DontCare; w.valid := false.B; w }
  def fromDecode(pc: UInt, inst: UInt): MicroOp = {
    val w = Wire(new MicroOp); w := DontCare
    w.valid := true.B; w.pc := pc; w.inst := inst; w.rd := inst(11,7); w
  }
}
val uopReg = RegInit(MicroOp.bubble)   // explicit、合法的 reset 值
```

## M3.6 測試與除錯 — ChiselSim ★版本重點

> 新專案用 **ChiselSim**。`ChiselTest` 已 deprecated（官方：*“not used or maintained by the core Chisel development team”*）。

### (A) 快速測試 — EphemeralSimulator
```scala
import chisel3._
import chisel3.simulator.EphemeralSimulator._
import org.scalatest.flatspec.AnyFlatSpec

class AdderSpec extends AnyFlatSpec {
  "Adder" should "add" in {
    simulate(new Adder) { dut =>
      dut.io.a.poke(3.U); dut.io.b.poke(5.U)
      dut.clock.step()
      dut.io.s.expect(8.U)
    }
  }
}
```

### (B) 測試套件 — ScalaTest trait（推薦）
```scala
import chisel3.simulator.scalatest.ChiselSim
import org.scalatest.flatspec.AnyFlatSpec
import org.scalatest.matchers.should.Matchers

class CounterSpec extends AnyFlatSpec with Matchers with ChiselSim {
  "Counter" should "count up" in {
    simulate(new MyCounter) { dut =>
      for (i <- 0 until 5) { dut.io.value.expect(i.U); dut.clock.step() }
    }
  }
}
```

| 動作 | 寫法 |
|------|------|
| 設定輸入 | `port.poke(v)` |
| 讀輸出 | `port.peek()`；BigInt 用 `.litValue` |
| 斷言 | `port.expect(v)` |
| 走時脈 | `dut.clock.step(n)` |
| 入口（快速 / 套件） | `EphemeralSimulator._` ／ `with ...scalatest.ChiselSim` |

### 波形（VCD）
```bash
sbt 'testOnly *CounterSpec -- -DemitVcd=1'     # 產 .vcd，用 GTKWave 開
```
> 其他格式經 `chisel3.simulator.scalatest.Cli` 的 `EmitFsdb` / `EmitVpd`。

### printf / println 不要混淆
```scala
println(s"Generating ALU width=$width")               // elaboration-time（生成期 log）
printf(p"pc=${Hexadecimal(pc)} inst=${Hexadecimal(inst)}\n")  // simulation runtime
```

### 🗄️ LEGACY — ChiselTest（**已 deprecated，僅供讀舊碼**）
```scala
import chiseltest._
class AdderSpecLegacy extends AnyFlatSpec with ChiselScalatestTester {
  "Adder" should "add" in { test(new Adder){ c => c.io.a.poke(3.U); c.clock.step(); c.io.s.expect(8.U) } }
}
```
| ChiselTest（舊） | ChiselSim（新） |
|------------------|-----------------|
| `test(new X){ c => }` | `simulate(new X){ dut => }` |
| `extends ChiselScalatestTester` | `with ChiselSim` / `import EphemeralSimulator._` |
| `WriteVcdAnnotation` | 旗標 `-DemitVcd=1` |
| poke/peek/expect/step | **相同** |

⚠️ **陷阱**：別在新專案 `import chiseltest._`；`peek()` 要數值用 `.litValue`；`step()` 無參數=1 拍。
