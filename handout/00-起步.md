# Part 0 — 起步：環境、Scala 速成、Hello Chisel

對應：書 §1、§10.1 ｜ docs: Installation / Getting Started

本章目標不是一次學完整個 Scala，也不是馬上寫出複雜硬體，而是建立最小閉環：
能編譯 Chisel、能產生 SystemVerilog、能跑一次模擬，並且知道錯誤訊息大概來自哪一層。
只要這個閉環成立，後面的每個概念都可以用「改一點、產生、測試、看波形」的方式驗證。

讀這章時要特別注意兩個時間點。Scala 程式在 elaboration-time 執行，負責產生硬體結構；
`Reg`、`when`、`Mux` 等 Chisel construct 產生的則是 runtime 硬體。很多新手錯誤都不是不懂語法，
而是把這兩個時間點混在一起。

**本章路線圖**

- 先學：固定工具鏈、最小 `build.sbt`、Scala 生成期與第一個 Module。
- 會踩坑：JDK/sbt/Chisel plugin 版本不合，以及把 Scala `if/for` 誤當硬體 runtime。
- 最後能做：產生 SystemVerilog、跑一次 ChiselSim，並知道錯誤大概來自 Scala、Chisel 或後端。

本章的驗收不是「看懂所有 Scala」，而是完成一個最小閉環：

| 步驟 | 成功時你應該能說清楚 |
|------|----------------------|
| compile | 錯誤是 Scala type error、Chisel elaboration error，還是 backend error |
| emit SystemVerilog | 哪些 `Reg` / `IO` 出現在產物裡 |
| run simulation | 測試在哪個 cycle poke / expect / step |
| 改一點再跑 | 修改是生成期參數改變，還是 runtime 硬體行為改變 |

## M0.1 環境與工具鏈

**一句話**：把 Chisel 7 跑起來、產生 Verilog、看波形。

工具鏈是教學的第一個風險點。Chisel 專案牽涉 JDK、Scala、sbt、Chisel plugin、CIRCT/firtool
與模擬器；任何一層版本不合，都可能讓初學者誤以為自己寫錯 RTL。建議一開始只用固定版本，
等能穩定跑完 `sbt test` 與 `emitSystemVerilog` 之後，再討論升級。

### 最小骨架 — `build.sbt`（Chisel 7.13.0）

```scala
// 標記：教學骨架。完整可跑設定見 ../examples/build.sbt。
scalaVersion := "2.13.18"
val chiselVersion = "7.13.0"

addCompilerPlugin("org.chipsalliance" % "chisel-plugin" % chiselVersion cross CrossVersion.full)
libraryDependencies += "org.chipsalliance" %% "chisel"    % chiselVersion
libraryDependencies += "org.scalatest"     %% "scalatest" % "3.2.19" % Test

scalacOptions ++= Seq("-language:reflectiveCalls", "-deprecation", "-feature", "-Xcheckinit")
```

> 最快路線：先跑本 repo 的 [`../examples/`](../examples/)；想從空白專案開始時，再 clone 官方範本 `git clone https://github.com/chipsalliance/chisel-template.git`。

### 語法速查 — 工具鏈

| 工具 | 用途 | 指令 |
|------|------|------|
| JDK 8–21 | 執行環境 | `java -version` |
| sbt | 建置/測試 | `sbt test`、`sbt run` |
| **Scala CLI** | 單檔快速試（新手推薦） | `scala-cli run Hello.scala` |
| GTKWave | 看波形 VCD | `gtkwave test.vcd` |
| IntelliJ / VSCode(Metals) | IDE | 匯入 `build.sbt` |

### Scala CLI 單檔試法（免建 sbt 專案）

```scala
// 標記：教學骨架。用來展示單檔 elaboration，不是本 repo 的主要範例入口。
//> using scala "2.13.18"
//> using dep "org.chipsalliance::chisel:7.13.0"
//> using plugin "org.chipsalliance:::chisel-plugin:7.13.0"

import chisel3._
import circt.stage.ChiselStage

class AndGate extends Module {
  val io = IO(new Bundle {
    val a = Input(Bool()); val b = Input(Bool()); val y = Output(Bool())
  })
  io.y := io.a & io.b
}
object Main extends App {
  println(ChiselStage.emitSystemVerilog(new AndGate))
}
```

⚠️ **陷阱**：JDK 版本 > 21 可能與 sbt/Scala 不相容；卡住先確認 `java -version` 落在 8–21。

完成本節後，你應該可以回答三件事：專案用哪個 Scala/Chisel 版本、Chisel plugin 是否有載入、
以及產生 SystemVerilog 的入口在哪裡。若其中一項不清楚，後面遇到 compiler error 會很難定位。

## M0.2 給硬體人的 Scala 速成 ★關鍵章

**一句話**：只學「描述硬體會用到」的 Scala，其餘略過。

> **心智模型**：你寫的 Scala 是「蓋電路的程式」。它在**生成期**執行，產出 RTL。
> 因此 Scala 的值/迴圈 = 生成期的「圖紙指令」；Chisel 的 `Reg`/`when` = 真正的硬體。

硬體人學 Scala 不需要先理解完整語言生態。這裡只需要掌握「用程式產生結構」：
用 `for` 複製 N 份電路、用 `if` 選擇要不要生成某個模組、用 `case class` 管理設計參數、
用 `map/reduce` 把一串硬體節點接成規律結構。這些都是 elaboration-time 的動作。

判斷標準很簡單：如果條件是 Scala `Int`、`Boolean` 或 `case class` 裡的參數，通常是在決定硬體長相；
如果條件是 `Bool`、`UInt`，通常是在描述已經存在的硬體訊號如何流動。

### 最小骨架 — 七個你會用到的 Scala 概念

```scala
// 標記：概念示意。這些是 Scala 生成期語法，不會直接長成 runtime 硬體。
// 1. val（不可變，硬體裡幾乎都用它） / var（可變，少用）
val width = 8
var i = 0

// 2. def（函數）— 之後拿來「生成硬體」
def add(a: Int, b: Int): Int = a + b

// 3. class / object / extends —— Module 就是 class
class Foo extends Module { /* ... */ }
object Bar { val MAX = 255 }          // object = 單例（放常數/工廠）

// 4. type inference（型別可省略）
val x = 3            // Scala 推斷為 Int

// 5. for / if（生成期控制流，不是硬體！）
for (i <- 0 until 4) { /* 重複生成 4 份硬體 */ }
val y = if (width > 4) "big" else "small"   // if 是表達式，有回傳值

// 6. 集合與高階函數（生成器核心）
val xs = Seq(1, 2, 3, 4)
val doubled = xs.map(_ * 2)               // Seq(2,4,6,8)
val total   = xs.reduce(_ + _)            // 10
xs.zipWithIndex.foreach { case (v, idx) => println(s"$idx -> $v") }

// 7. case class（之後參數化 config 用）
case class Config(width: Int, depth: Int)
val cfg = Config(width = 8, depth = 16)
```

### 語法速查 — Scala ↔ 硬體對照

| Scala 構件 | 何時用 | 對應硬體意義 |
|------------|--------|--------------|
| `val` | 命名一條線/一個常數 | wire 或 generate-time 常數 |
| `def` | 包裝可重用的硬體片段 | 「函數回傳硬體」= 生成器 |
| `for` / `if` | 重複/條件式**生成**結構 | 展開成多份硬體（非 runtime） |
| `.map`/`.reduce`/`.fold` | 由集合生成硬體陣列/樹 | mux tree、adder tree… |
| `case class` | 模組參數打包 | 設計參數（width/depth…） |
| `Option`/`Some`/`None` | 可選 port | optional port（見 Part 4） |

### Scala 語法拆解：先看得懂，再談優雅

Chisel 教學很容易太快跳到「這是 Scala idiom」，但硬體讀者真正需要的是先能拆開語法。
下面幾個語法會在 Part 4 之後反覆出現。

#### `case class`：一包不可變參數

```scala
case class CacheParams(nSets: Int, nWays: Int, blockBytes: Int = 64)

val p0 = CacheParams(64, 4)                         // positional arguments
val p1 = CacheParams(nSets = 64, nWays = 4)          // named arguments，較清楚
val p2 = p1.copy(nWays = 8)                          // 複製一份，只改 nWays
```

`case class` 很適合硬體參數，因為它自動提供 constructor、欄位 accessor、`copy`、`toString`、比較等功能。
對 generator 來說，這代表一組 config 可以被清楚命名、複製、列印與 sweep。

| 語法 | 意義 |
|------|------|
| `case class X(a: Int)` | 定義一種不可變資料包 |
| `X(a = 3)` | 用欄位名稱建立，避免參數順序看錯 |
| `p.a` | 讀欄位 |
| `p.copy(a = 4)` | 由舊 config 派生新 config |
| `blockBytes: Int = 64` | default argument |

#### `Option`：可能有，也可能沒有

```scala
val maybeDebug: Option[Int] = Some(32)
val noDebug: Option[Int] = None

maybeDebug.foreach { width =>
  println(s"debug width = $width")
}
```

`Option[T]` 的意思是「可能有一個 `T`」。在 Chisel generator 中，它常用來表達 optional hardware：
`Some(x)` 表示存在，`None` 表示完全不存在。這比用 `null` 安全，也比多個 Boolean 更容易追。

#### `match` / `case`：依照選項生成不同硬體

```scala
sealed trait MulKind
case object NoMul extends MulKind
case class IterMul(cycles: Int) extends MulKind

val mulKind: MulKind = IterMul(cycles = 4)
mulKind match {
  case NoMul       => println("no multiplier")
  case IterMul(n)  => println(s"iterative multiplier, cycles=$n")
}
```

`sealed trait` + `case object` + `case class` 常被稱為 ADT（algebraic data type）。它適合表示「互斥選項」：
沒有乘法器、迭代乘法器、pipeline 乘法器等。這種寫法比 `useMul`、`useFastMul`、`mulCycles`
三個欄位更不容易出現矛盾 config。

#### `_`：短函數參數

```scala
val xs = Seq(1, 2, 3)
xs.map(x => x * 2)     // 完整寫法
xs.map(_ * 2)          // 短寫：_ 代表目前元素
xs.reduce((a, b) => a + b)
xs.reduce(_ + _)       // 兩個 _ 分別代表 a, b
```

在 Chisel 裡看到 `reduce(_ +& _)`，先把它還原成 `(a, b) => a +& b`，再思考會生成什麼硬體。

#### `implicit`：省略傳遞的上下文

```scala
def emitName(x: String)(implicit prefix: String): String = prefix + x

implicit val p: String = "Core_"
emitName("Small")          // 等同 emitName("Small")(p)
```

`implicit` 不是硬體功能，而是 Scala 幫你自動補參數的機制。Rocket/Chipyard 用它傳遞全系統 `Parameters`。
好處是深層 module 不用每層手動傳 config；壞處是參數來源比較隱形。Part 10 會專門拆解。

#### 多組參數列表：`f(a)(b)`

```scala
def connect(width: Int)(name: String): String = s"$name:$width"
connect(32)("data")

class Foo(width: Int)(implicit p: String)
```

Scala function/class 可以有多組參數列表。Chisel/Chipyard 常用它來把「普通參數」和「隱含上下文」
分開：第一組是你明確傳的 `width/address/gen`，第二組可能是 `implicit p: Parameters`。
讀 `class Timer(params: TimerParams)(implicit p: Parameters)` 時，就把它拆成兩段：
先傳 Timer 自己的參數，再由 Scala 自動補全系統 config。

#### Tuple 與 pattern matching：一次回傳/拆開多個值

```scala
val pair = (3, "hit")          // type: (Int, String)
val (id, name) = pair          // id = 3, name = "hit"
val (_, wrap) = Counter(true.B, 10)

pair match {
  case (i, s) => println(s"$i $s")
}
```

`_` 在 tuple pattern 裡代表「我不需要這個欄位」。Chisel 常見 `val (_, wrap) = Counter(...)`，
意思是 Counter 回傳兩個值，但這裡只關心第二個 `wrap`。在 `foreach { case (rd, data) => ... }`
中，`case (rd, data)` 則是把 tuple 拆成兩個名字。

#### Scala collection：`Seq` / `List` / `Array` 不是硬體陣列

```scala
val xs = Seq(1, 2, 3)
val zeros = Seq.fill(4)(0.U(8.W))
val lanes = Seq.tabulate(4)(i => io.in(i) + 1.U)
val vec = VecInit(lanes)
```

`Seq`、`List`、`Array` 都是 Scala collection，存在於 elaboration-time；`Vec` 才是 Chisel 硬體 aggregate。
因此 `xs(i)` 的 `i` 必須是 Scala `Int`；`vec(idx)` 的 `idx` 可以是硬體 `UInt`。

| 寫法 | 層級 | 常見用途 |
|------|------|----------|
| `Seq(...)` | Scala 生成期集合 | 收集一批 wire/module，配合 `map/reduce` |
| `Seq.fill(n)(x)` | Scala 生成期重複 | 生成 n 個相似元素 |
| `Seq.tabulate(n)(i => ...)` | Scala 生成期 indexed generate | 依 index 生成規律硬體 |
| `Array(...)` | Scala/JVM 可變陣列 | 較少直接用，常見於舊 API 或 options |
| `Vec(...)` / `VecInit(...)` | Chisel 硬體 aggregate | 可被硬體 index、會進 RTL |

#### Scala number vs Chisel literal

```scala
val depth: Int = 1024
val base: BigInt = 0x80000000L

val idx = UInt(log2Ceil(depth).W)       // width 由 Scala Int 算出
io.addr := base.U(32.W)                 // Scala BigInt/Long 轉 Chisel UInt literal
io.mask := "b1111".U                    // binary literal
```

Scala 的 `Int`/`BigInt` 是生成期數字；Chisel 的 `UInt`/`SInt` 是硬體值或 literal。
位寬 `.W` 也需要 Scala 整數。address map、memory size、config 常用 `BigInt`，因為 SoC address
可能超過 32-bit signed `Int` 的範圍。

| 你看到的型別 | 層級 | 用途 |
|--------------|------|------|
| `Int` | Scala | width、depth、loop bound、小參數 |
| `BigInt` | Scala | address、large mask、config metadata |
| `Bool` / `UInt` / `SInt` | Chisel | 硬體 signal 或 literal |
| `.U` / `.S` | Scala literal 轉 Chisel literal | `3.U`、`(-1).S` |
| `.W` | Scala Int 轉 Chisel width | `32.W` |

⚠️ **新手三大陷阱**
1. Scala `==`（值比較，生成期） **≠** Chisel `===`（硬體相等比較）。
2. Scala `if`（生成期決定電路）**≠** `when`（runtime 的硬體 mux）。
3. `for (i <- 0 until n)` 不是硬體迴圈，是「生成 n 份硬體」。

對應：書 §10.1「A Little Bit of Scala」（本 Handout 前移到此）。

## M0.3 Hello Chisel

**一句話**：寫第一個 Module、blinking LED、看到生成的 Verilog、跑一次 ChiselSim。

第一個模組不要追求功能複雜，而要追求完整流程。`Blinky` 足夠小，卻包含了輸出 IO、
reset 初值、counter、暫存器更新與 SystemVerilog 產生。它是檢查工具鏈與心智模型的好例子：
`freq` 是生成期參數，`led` 是硬體暫存器，`wrap` 是每拍計算出來的硬體條件。

### 最小骨架 — Blinking LED

```scala
import chisel3._
import chisel3.util.Counter

class Blinky(freq: Int, startOn: Boolean = false) extends Module {
  val io = IO(new Bundle { val led = Output(Bool()) })

  val led = RegInit(startOn.B)
  val (_, wrap) = Counter(true.B, freq / 2)   // freq/2 拍翻轉一次 → 1 Hz
  when(wrap) { led := ~led }
  io.led := led
}
```

### 產生 Verilog / SystemVerilog

```scala
import circt.stage.ChiselStage
object Main extends App {
  println(ChiselStage.emitSystemVerilog(
    new Blinky(1000),
    firtoolOpts = Array("-disable-all-randomization", "-strip-debug-info")
  ))
}
```

### 第一個 ChiselSim 測試（細節見 Part 3）

```scala
import chisel3._
import chisel3.simulator.EphemeralSimulator._
import org.scalatest.flatspec.AnyFlatSpec

class BlinkyTest extends AnyFlatSpec {
  "Blinky" should "toggle" in {
    simulate(new Blinky(4)) { dut =>
      dut.clock.step(10)        // 跑 10 個 cycle
    }
  }
}
```

### 語法速查 — Module 解剖

| 元素 | 寫法 | 說明 |
|------|------|------|
| 宣告模組 | `class X extends Module` | 一個硬體模組 |
| 介面 | `val io = IO(new Bundle{...})` | 對外接腳 |
| 輸入/輸出 | `Input(T)` / `Output(T)` | 方向 |
| 隱含訊號 | clock / reset | **自動產生**，免宣告 |
| 連線 | `io.led := led` | `:=` 由右接到左 |

⚠️ **陷阱**：Chisel module **不需**手寫 `clock`/`reset`；`Reg` 系列會自動接上。看生成的 Verilog 會發現它們被隱含加入。

**練習**
- ★ 改 `Blinky` 讓 LED「每秒亮 200 ms」。完成標準：用兩個 threshold 表達亮暗時間，而不是只翻轉狀態。
- ★★ 用 `emitSystemVerilog` 觀察生成的 `clock`/`reset` 如何出現在 Verilog。完成標準：能指出哪個暫存器對應原本的 `led`。
- ★★ 在 `examples/` 中跑 `sbt "runMain handout.Main"`，確認產物能被 firtool 接受；若失敗，先分辨是 Scala 編譯錯還是 FIRRTL/CIRCT 錯。
