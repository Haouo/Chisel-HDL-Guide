# Part 0 — 起步：環境、Scala 速成、Hello Chisel

對應：書 §1、§10.1 ｜ docs: Installation / Getting Started

本章目標不是一次學完整個 Scala，也不是馬上寫出複雜硬體，而是建立最小閉環：
能編譯 Chisel、能產生 SystemVerilog、能跑一次模擬，並且知道錯誤訊息大概來自哪一層。
只要這個閉環成立，後面的每個概念都可以用「改一點、產生、測試、看波形」的方式驗證。

讀這章時要特別注意兩個時間點。Scala 程式在 elaboration-time 執行，負責產生硬體結構；
`Reg`、`when`、`Mux` 等 Chisel construct 產生的則是 runtime 硬體。很多新手錯誤都不是不懂語法，
而是把這兩個時間點混在一起。

**本章路線圖**

- 先學：固定工具鏈、最小 `build.sbt`、Scala 生成期與第一個 Module。
- 會踩坑：JDK/sbt/Chisel plugin 版本不合，以及把 Scala `if/for` 誤當硬體 runtime。
- 最後能做：產生 SystemVerilog、跑一次 ChiselSim，並知道錯誤大概來自 Scala、Chisel 或後端。

## M0.1 環境與工具鏈

**一句話**：把 Chisel 7 跑起來、產生 Verilog、看波形。

工具鏈是教學的第一個風險點。Chisel 專案牽涉 JDK、Scala、sbt、Chisel plugin、CIRCT/firtool
與模擬器；任何一層版本不合，都可能讓初學者誤以為自己寫錯 RTL。建議一開始只用固定版本，
等能穩定跑完 `sbt test` 與 `emitSystemVerilog` 之後，再討論升級。

### 最小骨架 — `build.sbt`（Chisel 7.13.0）

```scala
scalaVersion := "2.13.18"
val chiselVersion = "7.13.0"

addCompilerPlugin("org.chipsalliance" % "chisel-plugin" % chiselVersion cross CrossVersion.full)
libraryDependencies += "org.chipsalliance" %% "chisel"    % chiselVersion
libraryDependencies += "org.scalatest"     %% "scalatest" % "3.2.19" % Test

scalacOptions ++= Seq("-language:reflectiveCalls", "-deprecation", "-feature", "-Xcheckinit")
```

> 最快路線：直接 clone 官方範本 `git clone https://github.com/chipsalliance/chisel-template.git`。

### 語法速查 — 工具鏈

| 工具 | 用途 | 指令 |
|------|------|------|
| JDK 8–21 | 執行環境 | `java -version` |
| sbt | 建置/測試 | `sbt test`、`sbt run` |
| **Scala CLI** | 單檔快速試（新手推薦） | `scala-cli run Hello.scala` |
| GTKWave | 看波形 VCD | `gtkwave test.vcd` |
| IntelliJ / VSCode(Metals) | IDE | 匯入 `build.sbt` |

### Scala CLI 單檔試法（免建 sbt 專案）

```scala
//> using scala "2.13.18"
//> using dep "org.chipsalliance::chisel:7.13.0"
//> using plugin "org.chipsalliance:::chisel-plugin:7.13.0"

import chisel3._
import circt.stage.ChiselStage

class AndGate extends Module {
  val io = IO(new Bundle {
    val a = Input(Bool()); val b = Input(Bool()); val y = Output(Bool())
  })
  io.y := io.a & io.b
}
object Main extends App {
  println(ChiselStage.emitSystemVerilog(new AndGate))
}
```

⚠️ **陷阱**：JDK 版本 > 21 可能與 sbt/Scala 不相容；卡住先確認 `java -version` 落在 8–21。

完成本節後，你應該可以回答三件事：專案用哪個 Scala/Chisel 版本、Chisel plugin 是否有載入、
以及產生 SystemVerilog 的入口在哪裡。若其中一項不清楚，後面遇到 compiler error 會很難定位。

## M0.2 給硬體人的 Scala 速成 ★關鍵章

**一句話**：只學「描述硬體會用到」的 Scala，其餘略過。

> **心智模型**：你寫的 Scala 是「蓋電路的程式」。它在**生成期**執行，產出 RTL。
> 因此 Scala 的值/迴圈 = 生成期的「圖紙指令」；Chisel 的 `Reg`/`when` = 真正的硬體。

硬體人學 Scala 不需要先理解完整語言生態。這裡只需要掌握「用程式產生結構」：
用 `for` 複製 N 份電路、用 `if` 選擇要不要生成某個模組、用 `case class` 管理設計參數、
用 `map/reduce` 把一串硬體節點接成規律結構。這些都是 elaboration-time 的動作。

判斷標準很簡單：如果條件是 Scala `Int`、`Boolean` 或 `case class` 裡的參數，通常是在決定硬體長相；
如果條件是 `Bool`、`UInt`，通常是在描述已經存在的硬體訊號如何流動。

### 最小骨架 — 七個你會用到的 Scala 概念

```scala
// 1. val（不可變，硬體裡幾乎都用它） / var（可變，少用）
val width = 8
var i = 0

// 2. def（函數）— 之後拿來「生成硬體」
def add(a: Int, b: Int): Int = a + b

// 3. class / object / extends —— Module 就是 class
class Foo extends Module { /* ... */ }
object Bar { val MAX = 255 }          // object = 單例（放常數/工廠）

// 4. type inference（型別可省略）
val x = 3            // Scala 推斷為 Int

// 5. for / if（生成期控制流，不是硬體！）
for (i <- 0 until 4) { /* 重複生成 4 份硬體 */ }
val y = if (width > 4) "big" else "small"   // if 是表達式，有回傳值

// 6. 集合與高階函數（生成器核心）
val xs = Seq(1, 2, 3, 4)
val doubled = xs.map(_ * 2)               // Seq(2,4,6,8)
val total   = xs.reduce(_ + _)            // 10
xs.zipWithIndex.foreach { case (v, idx) => println(s"$idx -> $v") }

// 7. case class（之後參數化 config 用）
case class Config(width: Int, depth: Int)
val cfg = Config(width = 8, depth = 16)
```

### 語法速查 — Scala ↔ 硬體對照

| Scala 構件 | 何時用 | 對應硬體意義 |
|------------|--------|--------------|
| `val` | 命名一條線/一個常數 | wire 或 generate-time 常數 |
| `def` | 包裝可重用的硬體片段 | 「函數回傳硬體」= 生成器 |
| `for` / `if` | 重複/條件式**生成**結構 | 展開成多份硬體（非 runtime） |
| `.map`/`.reduce`/`.fold` | 由集合生成硬體陣列/樹 | mux tree、adder tree… |
| `case class` | 模組參數打包 | 設計參數（width/depth…） |
| `Option`/`Some`/`None` | 可選 port | optional port（見 Part 4） |

⚠️ **新手三大陷阱**
1. Scala `==`（值比較，生成期） **≠** Chisel `===`（硬體相等比較）。
2. Scala `if`（生成期決定電路）**≠** `when`（runtime 的硬體 mux）。
3. `for (i <- 0 until n)` 不是硬體迴圈，是「生成 n 份硬體」。

對應：書 §10.1「A Little Bit of Scala」（本 Handout 前移到此）。

## M0.3 Hello Chisel

**一句話**：寫第一個 Module、blinking LED、看到生成的 Verilog、跑一次 ChiselSim。

第一個模組不要追求功能複雜，而要追求完整流程。`Blinky` 足夠小，卻包含了輸出 IO、
reset 初值、counter、暫存器更新與 SystemVerilog 產生。它是檢查工具鏈與心智模型的好例子：
`freq` 是生成期參數，`led` 是硬體暫存器，`wrap` 是每拍計算出來的硬體條件。

### 最小骨架 — Blinking LED

```scala
import chisel3._
import chisel3.util.Counter

class Blinky(freq: Int, startOn: Boolean = false) extends Module {
  val io = IO(new Bundle { val led = Output(Bool()) })

  val led = RegInit(startOn.B)
  val (_, wrap) = Counter(true.B, freq / 2)   // freq/2 拍翻轉一次 → 1 Hz
  when(wrap) { led := ~led }
  io.led := led
}
```

### 產生 Verilog / SystemVerilog

```scala
import circt.stage.ChiselStage
object Main extends App {
  println(ChiselStage.emitSystemVerilog(
    new Blinky(1000),
    firtoolOpts = Array("-disable-all-randomization", "-strip-debug-info")
  ))
}
```

### 第一個 ChiselSim 測試（細節見 Part 3）

```scala
import chisel3._
import chisel3.simulator.EphemeralSimulator._
import org.scalatest.flatspec.AnyFlatSpec

class BlinkyTest extends AnyFlatSpec {
  "Blinky" should "toggle" in {
    simulate(new Blinky(4)) { dut =>
      dut.clock.step(10)        // 跑 10 個 cycle
    }
  }
}
```

### 語法速查 — Module 解剖

| 元素 | 寫法 | 說明 |
|------|------|------|
| 宣告模組 | `class X extends Module` | 一個硬體模組 |
| 介面 | `val io = IO(new Bundle{...})` | 對外接腳 |
| 輸入/輸出 | `Input(T)` / `Output(T)` | 方向 |
| 隱含訊號 | clock / reset | **自動產生**，免宣告 |
| 連線 | `io.led := led` | `:=` 由右接到左 |

⚠️ **陷阱**：Chisel module **不需**手寫 `clock`/`reset`；`Reg` 系列會自動接上。看生成的 Verilog 會發現它們被隱含加入。

**練習**
- ★ 改 `Blinky` 讓 LED「每秒亮 200 ms」。完成標準：用兩個 threshold 表達亮暗時間，而不是只翻轉狀態。
- ★★ 用 `emitSystemVerilog` 觀察生成的 `clock`/`reset` 如何出現在 Verilog。完成標準：能指出哪個暫存器對應原本的 `led`。
- ★★ 在 `examples/` 中跑 `sbt "runMain handout.Main"`，確認產物能被 firtool 接受；若失敗，先分辨是 Scala 編譯錯還是 FIRRTL/CIRCT 錯。
