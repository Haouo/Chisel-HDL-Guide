# Part 7 — 記憶體、Clock/Reset、BlackBox（進階）

對應：書 §6.4、§11.4、Appendix A.2 ｜ `import chisel3._`、`import chisel3.util._`

> 進入「會接真實 SRAM macro、多時脈域、vendor IP」的領域。這些是 SoC 整合邊界的必備技巧。

---

## M7.1 `RawModule`：自己掌控 clock/reset

**一句話**：普通 `Module` 有 implicit clock/reset；`RawModule` 沒有，你自己接。

```scala
class AsyncResetWrapper extends RawModule {
  val clock = IO(Input(Clock()))
  val reset = IO(Input(AsyncReset()))
  val in    = IO(Input(UInt(32.W)))
  val out   = IO(Output(UInt(32.W)))
  withClockAndReset(clock, reset) {       // 在這個區塊內 Reg 才有 clock/reset
    out := RegNext(in)
  }
}
```
適合：top-level wrapper、multi-clock design、blackbox wrapper、ASIC 整合邊界、特殊 reset domain。

---

## M7.2 `withClockAndReset` 與 CDC 警告

```scala
withClockAndReset(cpuClock,  cpuReset)  { val core = Module(new Core) }
withClockAndReset(periClock, periReset) { val uart = Module(new Uart) }
```
> ⚠️ **重要**：Chisel **不會自動幫你做 CDC**。跨時脈域仍需：`AsyncQueue`、synchronizer、pulse crossing、gray-code FIFO、ready-valid CDC bridge。

## M7.3 `BlackBox` / `ExtModule`：接外部 Verilog / macro

**一句話**：包 SRAM macro、PLL、vendor IP、pad cell、手寫 Verilog。

```scala
class SramBlackBox extends BlackBox with HasBlackBoxResource {
  val io = IO(new Bundle {
    val clk  = Input(Clock()); val en = Input(Bool()); val wen = Input(Bool())
    val addr = Input(UInt(10.W)); val din = Input(UInt(32.W)); val dout = Output(UInt(32.W))
  })
  addResource("/sram_macro.v")            // 把 .v 一起送進產物
}
```
| 類別 | 差異 |
|------|------|
| `BlackBox` | 有 `val io = IO(new Bundle{...})`，name 對應外部 module |
| `ExtModule` | 直接宣告 port（無 `io` 包裝），更貼近 SystemVerilog |

⚠️ **陷阱**：port 名稱與型別必須與外部 Verilog **完全一致**；clock 顯式 `Input(Clock())`。

## M7.4 `SyncReadMem`：對齊讀延遲與 metadata

**一句話**：synchronous-read memory，cycle N 給 addr、cycle N+1 得 data；metadata 要一起延遲。

```scala
val ren   = io.req.fire
val raddr = io.req.bits.addr
val rdata = mem.read(raddr, ren)                 // 下一拍才有

val reqReg = RegEnable(io.req.bits, ren)         // ★ metadata 一起延遲對齊
io.resp.valid     := RegNext(ren, false.B)
io.resp.bits.addr := reqReg.addr
io.resp.bits.data := rdata
```
> Cache bug 常來自：tag/data read latency、miss detect latency、request metadata 沒對齊。

## M7.5 Memory wrapper：不要到處直接用 `SyncReadMem`

**一句話**：包一層，之後可替換成 BRAM / SRAM macro / sim model / masked-write / ECC 版。

```scala
class Sram1R1W[T <: Data](gen: T, depth: Int) extends Module {
  val aw = log2Ceil(depth)
  val io = IO(new Bundle {
    val r = new Bundle { val en = Input(Bool()); val addr = Input(UInt(aw.W)); val data = Output(gen) }
    val w = new Bundle { val en = Input(Bool()); val addr = Input(UInt(aw.W)); val data = Input(gen) }
  })
  val mem = SyncReadMem(depth, gen)
  when(io.w.en){ mem.write(io.w.addr, io.w.data) }
  io.r.data := mem.read(io.r.addr, io.r.en)
}
```

## M7.6 Read-under-write policy 要明確

**一句話**：同位址同拍讀寫，各 backend 行為不一致；要 write-first 就自己 bypass。

```scala
val raw    = mem.read(raddr, ren)
val bypass = wen && ren && (waddr === raddr)
val rdata  = Mux(bypass, wdata, raw)             // write-first
```

## M7.7 Register file 不一定要用 memory

```scala
val regs = RegInit(VecInit(Seq.fill(32)(0.U(32.W))))
val rs1  = Mux(io.rs1 === 0.U, 0.U, regs(io.rs1))   // x0 恆 0
val rs2  = Mux(io.rs2 === 0.U, 0.U, regs(io.rs2))
when(io.wen && io.rd =/= 0.U){ regs(io.rd) := io.wdata }
```
小型 CPU 用 `Vec[Reg]` 即可。大型才需考慮：multi-port cost、banking、SRAM macro、bypass、read latency、write-first。

---

## M7.8 ROM / 記憶體預載：`loadMemoryFromFileInline`

```scala
import chisel3.util.experimental.loadMemoryFromFileInline
val imem = SyncReadMem(depth, UInt(32.W))
loadMemoryFromFileInline(imem, "program.hex")
```
| 階段 | 載入方式 |
|------|----------|
| simulation | preload hex |
| FPGA | init file |
| ASIC | boot ROM macro / 外部記憶體 |

⚠️ **陷阱**：不要把「測試載入方式」綁死進硬體架構；用 wrapper/參數隔離。

**練習**
- ★ 把 Wildcat 的 `imem` 換成 `Sram1R1W` wrapper + `loadMemoryFromFileInline`。
- ★★ 為 `Sram1R1W` 加 write-first bypass，ChiselSim 驗證同位址讀寫。
- ★★★ 寫一個 `RawModule` top + 兩個時脈域，中間用 `AsyncQueue` 過 CDC。
